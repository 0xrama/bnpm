

This is the source code for Better NPM. We have barely started but the main point of the project is to have a better way to install npm packages with more details and also at the same time. when we use bnpmx understand what package we're installing and what permissions or inital scripts is this specific package running on the system. 
If the package you're installing as a very recent update the last hour or so. have an additional prompt saying this package has been updated very recently do you want to install this package? This is an extra step to ensure the most recent packages get a warning to check and avoid any supply chain attacks. 

The code has to be minimal and performant. The packages should be installed at a faster scale similar to how pnpm works. We're in the same league as PNPM we arent a run time like bun we're just a package manager with more details about the package and what could the package potentially do in the system. And if the recent versions have some dangerous commands or flows.
